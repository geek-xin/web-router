# Chunky 路由控制台设计说明

## 背景

`web-router` 当前管理后台已经具备路由配置、启停、复制、JSON 查看/编辑、请求日志和 SSE 实时日志能力。本次设计只调整前端管理后台的信息组织和视觉表达，不改变后端 API、路由配置模型、动态路由刷新机制或本地端口代理语义。

参考页面：

- `https://skills.yangsir.net/trending`
- `https://skills.yangsir.net/skill/find-skills`

参考图的核心视觉语言是 Memphis / chunky brutalism：粗黑描边、厚重偏移阴影、糖果色分区、超大圆角、榜单式卡片和强操作按钮。本项目需要吸收这种视觉语言，但仍保持 `PRODUCT.md` 中确认的产品定位：可信、克制、高效。

## 目标

1. 将管理后台首页改造成 “Chunky 路由控制台”：路由配置像卡片榜单一样易扫读，状态和风险更清楚。
2. 将单条路由详情改为右侧抽屉：既接近参考详情页的侧栏信息组织，又不离开当前单页管理后台。
3. 保留现有能力：新增、编辑、复制、删除、启停、访问、日志、JSON 查看/保存。
4. 保持后台任务效率：视觉更有辨识度，但不牺牲配置安全、可读性和排障速度。

## 非目标

- 不新增独立详情页。
- 不新增后端接口。
- 不改 `RouteConfig` / `RouteConfigDto` / JSON 文件格式。
- 不改变 Gateway 与本地端口代理转发语义。
- 不实现 SkillForge 的完整营销导航、横幅入口或浮动礼物组件。
- 不把装饰性玻璃态、花哨动效或营销站风格带入管理后台。

## 设计方向

采用已确认的 **方案 B：Chunky 路由控制台**。

### 首页

首页保留单页管理后台结构，重做为：

1. **顶部操作区**
   - 左侧：`Web Router` 品牌标识。
   - 中间：搜索框。
   - 右侧：刷新、新增路由、批量删除。
   - 风格：3px 黑色描边、厚重阴影、橙色主操作按钮、圆角胶囊按钮。

2. **主标题区**
   - 标题：`路由管理`
   - 描述：`路径转发、监听端口、实时日志，一处管理。`
   - 可带一个橙色图标块，借鉴参考图的热榜页标题。

3. **状态摘要**
   - 三张 chunky 数据卡：
     - 配置总数
     - 启用路由
     - 配置目录
   - 使用浅黄、薄荷绿、浅蓝/淡紫等低饱和糖果色。
   - 配置目录保留展开/隐藏完整路径能力。

4. **路由卡片墙**
   - 每条路由是一张榜单式卡片。
   - 顶部彩色条表达序号/状态：
     - 启用：薄荷绿、浅蓝、淡黄等轮换。
     - 停用：灰白底 + 明确 `已停用` 文案。
   - 主体展示：
     - 路由名称
     - 配置 ID / 文件名
     - 监听地址
     - 默认目标地址
     - 代理地址
     - 路径前缀标签
   - 底部操作：
     - 查看
     - 日志
     - 拷贝
     - 编辑
     - 访问
     - 删除
   - 禁用/不可访问状态必须有文字说明，不只靠颜色。

### 右侧详情抽屉

点击路由卡片的 `查看` 后，打开右侧详情抽屉，而不是只打开 JSON 弹窗。

抽屉内容：

1. **头部**
   - 路由名称
   - 运行状态徽章
   - 配置文件 ID / JSON 文件名
   - 关闭按钮

2. **关键数据块**
   - 监听地址
   - 默认目标地址
   - 代理地址
   - 路径前缀数量

3. **路径前缀**
   - 以 chunky 标签展示全部 `pathPrefixes`。
   - 空列表时展示 “未配置路径前缀，请求走默认地址”。

4. **原始 JSON**
   - 保留当前 JSON 查看/编辑能力。
   - JSON 区域使用等宽字体、浅色代码背景和粗描边容器。
   - 保存失败沿用现有 `fetchJson()` 业务错误语义。

5. **操作区**
   - 复制访问地址
   - 打开访问页
   - 编辑
   - 查看日志
   - 删除

6. **日志摘要入口**
   - 展示 “最近请求 / 错误 / 慢请求” 的摘要入口。
   - 详细实时日志仍使用现有 `RouteLogDialog`。

小屏策略：

- 右侧抽屉在窄屏改为全屏弹层。
- 操作按钮换行排列。
- JSON 区域横向滚动，不挤压正文。

## 视觉规范

### 色彩

使用浅底 chunky 风格，不沿用当前深色玻璃态作为主界面。

建议色彩角色：

- 主操作橙：用于新增、刷新、主要 CTA。
- 薄荷绿：表示启用、运行中、成功。
- 浅黄：表示配置、提示、复制。
- 浅蓝：表示信息、代理、日志。
- 粉色：表示选择、辅助分类。
- 风险红/珊瑚：表示删除、停用、错误。
- 黑色：用于粗描边、标题和主要文字。
- 近白/浅暖灰：用于页面背景和卡片主体。

规则：

- 黑色描边必须服务结构，不做无意义装饰。
- 状态色必须配合文字标签。
- 删除、停用、错误不能与普通标签混用同色。

### 形状与阴影

- 卡片和主容器：20–24px 圆角。
- 小按钮/标签：999px 胶囊或 14–18px 圆角。
- 主描边：3px 黑色。
- 次描边：2px 黑色或低透明黑色。
- 主阴影：黑色偏移厚阴影，类似 `6px 8px 0 #000`。
- 悬浮态：轻微上移或阴影加深。
- 按下态：回到原位，形成 “按下去” 的触感。

### 排版

- 保留 Inter / 系统 sans 作为主体字体，确保中文可读。
- 标题使用 800–900 字重。
- 标签和按钮使用粗体，但避免所有文字都超粗造成噪音。
- 配置 ID、JSON、地址使用等宽字体。

### 动效

- 只使用 150–220ms 的交互反馈。
- 允许 hover 上浮、按下回弹、抽屉滑入。
- 禁止复杂入场动画、弹跳、过度装饰动效。
- 必须保留 `prefers-reduced-motion` 兜底。

## 组件设计

### 顶部工具栏

职责：

- 提供全局搜索、刷新、新增、批量删除。
- 让用户随时知道当前在 “路由控制台”。

行为：

- 搜索保持当前按名称、目标地址、监听地址、路径前缀过滤。
- 批量删除无选中项时禁用。
- 刷新沿用当前 `loadRoutes()`。

### 状态摘要卡

职责：

- 快速展示配置规模和启用状态。
- 暴露配置目录。

行为：

- 配置目录仍可切换显示完整路径。
- 数字要醒目，但不要遮盖管理操作。

### 路由卡片

职责：

- 首页主要工作单元。
- 一眼看清 “这条路由是否启用、监听哪里、转发到哪里、有哪些前缀”。

字段：

- `route.name`
- `route.id`
- `targetUrl`
- `accessPageBaseUrl`
- `localIp` / `localPort`
- `enabled`
- `pathPrefixes`

操作：

- `查看`：打开右侧详情抽屉。
- `日志`：打开现有日志弹窗。
- `拷贝`：进入现有 copy 流程。
- `编辑`：进入现有编辑表单。
- `访问`：沿用当前 `routeAccessUrl()`。
- `删除`：沿用当前删除确认。

### 路由详情抽屉

职责：

- 替代当前单纯 JSON 查看弹窗，成为单条路由的详情面板。
- 聚合配置、JSON、操作和日志入口。

状态：

- 打开/关闭。
- JSON 加载中。
- JSON 保存中。
- JSON 解析错误。
- API 保存错误。

约束：

- 不改变现有 JSON 保存 API。
- JSON 编辑能力必须保留。
- 删除必须仍经过确认，不在抽屉内直接无确认删除。

### 日志弹窗

职责保持不变：

- 实时日志。
- 总耗时 Top。
- 慢请求。
- 诊断分析。

视觉可同步改为 chunky 外观，但不改变数据逻辑。

## 数据流

### 首页加载

1. `App` 调用 `/admin/api/routes`。
2. 保存到 `routes`。
3. 搜索过滤得到 `filteredRoutes`。
4. 渲染状态摘要和路由卡片墙。

### 查看详情

1. 用户点击路由卡片 `查看`。
2. `App` 记录 `detailRoute`。
3. 调用 `/admin/api/routes/{id}/raw` 获取 JSON。
4. 打开右侧详情抽屉。
5. 用户保存 JSON 时，沿用 `PUT /admin/api/routes/{id}`。
6. 保存成功后刷新路由列表并关闭或更新抽屉内容。

### 访问和日志

- `访问` 沿用当前 `openAccess(route)`。
- `日志` 沿用当前 `RouteLogDialog`。

## 错误处理

- 路由加载失败：toast 错误。
- JSON 加载失败：抽屉内显示错误，同时 toast。
- JSON 解析失败：阻止提交，显示 “JSON 格式无效”。
- 后端业务错误：沿用 `fetchJson()` 提取 `message`。
- 不可访问路由：按钮禁用并显示 title/提示文案。
- 删除操作：必须走确认弹窗。

## 可访问性

- 所有按钮保留可读 `aria-label`。
- 抽屉打开后焦点进入抽屉，关闭后回到触发按钮。
- 抽屉支持 Esc 关闭。
- 焦点态使用清晰黑色/橙色描边或 ring。
- 状态必须有文字标签，不只靠颜色。
- 小屏全屏抽屉必须可滚动。
- 支持 `prefers-reduced-motion`。

- `cd frontend && npm test -- --run`
- `cd frontend && npm run build`

### 手动验收

1. 打开 `/admin`。
2. 路由列表加载成功。
3. 搜索仍能按路由名、地址、前缀过滤。
4. 新增、编辑、复制、删除、启停路由仍可用。
5. 点击 `查看` 打开右侧详情抽屉。
6. 抽屉中 JSON 可查看、编辑、保存。
7. 点击 `日志` 仍打开日志弹窗。
8. 点击 `访问` 按原逻辑打开目标访问页。
9. 小屏下卡片单列、详情全屏可滚动。
10. 禁用路由和危险操作有明确文案，不只靠颜色区分。

## 实现边界

优先修改 `frontend/`：

- `frontend/src/App.tsx`
- `frontend/src/styles.css`
- `frontend/src/components/ui/*`
- `frontend/src/features/routes/*`
- `frontend/src/features/logs/RouteLogDialog.tsx`（只做外观同步时修改）

构建后同步产物：

- `src/main/resources/static/admin/assets/app.css`
- `src/main/resources/static/admin/assets/app.js`

不修改：

- 后端服务、控制器、实体、DTO。
- `config/routes` 运行时数据目录。
- Spring Cloud Gateway 和本地端口代理逻辑。
